# Nocturne · A Timeline Museum of Art History

An interactive, fully static, browser-based museum. The entry point is a dark
constellation star-map of art history — 19 periods and 70+ artists placed by
their real dates. Zoom into a period to reveal its artists, open an artist's
placard, then step into a first-person walkable 3D gallery hung with their
most famous paintings, every one sourced from Wikipedia / Wikidata /
Wikimedia Commons.

No database. No backend. No API keys. Hostable on GitHub Pages.

## Running locally

```bash
npm install
npm run dev      # http://localhost:5173
```

## Building the static site

```bash
npm run build    # outputs ./dist — a fully static site
npm run preview  # serve the production build locally
```

Asset URLs are relative (`base: './'` in [vite.config.ts](vite.config.ts)),
so the same build works at `https://<user>.github.io/<repo>/` or any static host.

## Deploying to GitHub Pages

1. Push this project to a GitHub repository (default branch `main`).
2. In the repo: **Settings → Pages → Build and deployment → Source: "GitHub Actions"**.
3. Push to `main`. The included workflow
   [.github/workflows/deploy.yml](.github/workflows/deploy.yml) builds the site
   and publishes `dist/` to Pages. No secrets are required.

Manual alternative (no Actions): `npm run build`, then publish the `dist/`
folder with any static-hosting method (e.g. the `gh-pages` branch).

## Where the data comes from

Everything factual — biographies, portraits, artwork titles, dates, mediums,
collections, stories, and images — comes from **Wikipedia, Wikidata, and
Wikimedia Commons**. Nothing is invented; missing fields render as honest
"information unavailable" notes.

The data ships as a static cache in [src/data](src/data)
(`periods.json`, `artists.json`, `artworks.json`), generated by:

```bash
npm run fetch-data   # regenerates src/data/*.json from live wiki APIs
```

The pipeline ([scripts/fetchWikiData.mjs](scripts/fetchWikiData.mjs)) reads the
hand-curated registry of periods/artists in
[scripts/curated.mjs](scripts/curated.mjs), then per artist:

1. **Wikipedia REST** — biography extract, portrait, Wikidata QID
2. **Wikidata SPARQL** — the artist's works ranked by fame (sitelink count),
   with inception year, medium, and collection
3. **Wikipedia REST** — per-artwork story, when an English article exists
4. **Commons imageinfo** — exact thumbnail URLs, dimensions, license + credit

Only images hosted on Wikimedia Commons (freely licensed / public domain) are
used. Works whose images are rights-restricted (much of Surrealism, Abstract
Expressionism, and Pop Art) appear as sourced metadata placards in the
galleries instead — by design, never a fabricated image.

**To add artists or periods:** edit [scripts/curated.mjs](scripts/curated.mjs)
and run `npm run fetch-data`.

Attribution: article text CC BY-SA 4.0 via Wikipedia; each image's license and
Commons file page are linked in the painting inspect view.

## Controls

| Where | Input |
| --- | --- |
| Timeline | drag to pan · scroll/pinch to zoom · arrows / `+` `-` keys · click stars |
| Gallery | click to capture the mouse · WASD/arrows to walk · `Shift` to stride · `Esc` to release |
| Gallery (touch) | drag to look · on-screen pad to walk · tap a painting |
| Anywhere | `Esc` closes cards and overlays |

A **Quality** toggle (bottom-right in galleries) switches off reflections,
shadows, and high-DPI rendering for modest hardware.

## QA

Automated end-to-end suite (Playwright, runs against the production build):

```bash
npm run build
npx playwright install chromium   # first time only
npm run test:e2e
```

The suite covers: timeline load, zoom/pan, period filter, artist card,
gallery entry, all 8 paintings mounting, inspect open/close, return to
timeline, and zero uncaught console errors.
